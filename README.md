# Semantic Search — Hands-On Large Language Models

A fully local semantic search engine and RAG system built on top of the **Hands-On Large Language Models** book by Jay Alammar.

Upload the PDF once, then either search for the most relevant passages or ask a question and get a full generated answer — all running locally with no API keys required.

---

## What This Project Does

| Feature | Description |
|---------|-------------|
| **PDF Ingestion** | Upload the book PDF and it gets extracted, chunked, embedded, and stored |
| **Semantic Search** | Find the most relevant passages using vector similarity |
| **Agentic RAG** | Ask a question and get a full answer generated by a local LLM |
| **Gradio UI** | Clean web interface with two search modes |
| **REST API** | Full FastAPI backend with Swagger docs |

## How It Works

### Ingestion Pipeline

```
PDF File
    |
    v  PyMuPDF + UTF-8 cleaning
Extract text per page
    |
    v  ingest_controller
Split into chunks (size=500, overlap=50)
    |
    v  HuggingFace all-MiniLM-L6-v2
Generate embedding vectors (batches of 90)
    |
    v  ChromaDB
Store vectors + text + metadata (page_number, source)
```

### Semantic Search Pipeline

```
User Query
    |
    v  HuggingFace all-MiniLM-L6-v2
Generate query vector
    |
    v  ChromaDB cosine similarity search
Top 5 most relevant chunks
    |
    v  API Response
text + page_number + score
```

### Agentic RAG Pipeline (LangGraph)

```
User Question
    |
    v  Retrieve
ChromaDB similarity search → top 5 chunks
    |
    v  Grade Documents
llama3.2 checks relevance of each chunk (yes/no)
    |
    v  Generate
llama3.2 generates answer from relevant chunks only
    |
    v  Hallucination Grader
llama3.2 checks answer is grounded in the retrieved facts
    |
    v  Answer Grader
llama3.2 checks answer actually resolves the question
    |
    v  Final Answer
```

## Getting Started

### Option 1 — Run Locally

**Requirements:** Python 3.11, Conda, Ollama installed on your machine.

**Step 1 — Clone the repo**
```bash
git clone https://github.com/Huda-Mawood/Semantic-Search.git
cd Semantic-Search
```

**Step 2 — Create and activate environment**
```bash
conda create -n semantic-search python=3.11
conda activate semantic-search
pip install -r requirements.txt
```

**Step 3 — Pull and run Ollama model**
```bash
ollama pull llama3.2
ollama serve
```

**Step 4 — Start the server**
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

**Step 5 — Open in browser**
```
http://localhost:8000/ui      ← Gradio UI
http://localhost:8000/docs    ← Swagger API docs
```

---

### Option 2 — Run with Docker Compose

**Requirements:** Docker and Docker Compose installed.

```bash
git clone https://github.com/Huda-Mawood/Semantic-Search.git
cd Semantic-Search
docker-compose up --build
```

Everything starts automatically — the app, Ollama, and the model download.

```
http://localhost:8000/ui
http://localhost:8000/docs
```

To stop:
```bash
docker-compose down
```

To stop and remove all data:
```bash
docker-compose down -v
```

### Health Check
```http

```

---

### Upload PDF
```http
POST /api/v1/upload-pdf
Content-Type: multipart/form-data
```

**Postman:**
```
Body → form-data
Key:   file    [File]
Value: select your PDF
```

**Response:**
```json
{
    "message": "PDF ingested successfully.",
    "filename": "book.pdf",
    "total_chunks": 1725
}
```

---

### Semantic Search
```http
POST /api/v1/search
Content-Type: application/json
```

**Postman:**
```
Body → raw → JSON
```

```json
{ "query": "What is the attention mechanism?" }
```

**Response:**
```json
{
    "query": "What is the attention mechanism?",
    "total_results": 5,
    "results": [
        {
            "text": "The attention mechanism is a way to score...",
            "page_number": 137,
            "source": "./data/temp/book.pdf",
            "score": 0.4882
        }
    ]
}
```

---

**Postman:**

## Configuration

All settings live in `config.py` and can be overridden in `.env`:

| Variable | Default | Description |
|----------|---------|-------------|
| `EMBEDDING_MODEL_NAME` | `all-MiniLM-L6-v2` | HuggingFace embedding model |
| `OLLAMA_MODEL` | `llama3.2` | Ollama LLM model |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama server URL |
| `CHUNK_SIZE` | `500` | Characters per chunk |
| `CHUNK_OVERLAP` | `50` | Overlap between chunks |
| `TOP_K_RESULTS` | `5` | Number of results returned |
| `CHROMA_COLLECTION_NAME` | `llm_book` | ChromaDB collection name |
| `CHROMA_DB_PATH` | `./data/chroma_db` | ChromaDB storage path |

## Notes

- The PDF is deleted from temp storage after ingestion
- ChromaDB stores data locally — no external service needed
- Re-uploading the same PDF adds duplicate chunks — clear `data/chroma_db/` first
- Make sure Ollama is running before using the RAG endpoint
- The embedding model downloads automatically on first run (~90MB)
